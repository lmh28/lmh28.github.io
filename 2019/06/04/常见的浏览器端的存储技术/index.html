常见的浏览器端的存储技术

常见的浏览器端存储技术有三种

  • localStorage
  • cookie
  • sessionStorage

共同点

都是保存在浏览器端，且同源

cookie 和 sessionStorage 以及 localStorage 的区别是什么？

  1. 从生命周期来说

    • Cookie 作用范围。如不设置失效时间，则表示这个 cookie 的生命周期为浏览器回话期间，关闭浏览器，cookie 就会消失。若设置了失效时间，浏览器会把 cookie 保存到硬盘中，关闭后再打开浏览器这些 cookie 仍然有效直到超过设定的过期时间。
    • localStorage 除非被清除，否则始终有效，窗口或浏览器关闭也一直保存，因此用作持久数据
    • sessionStorage 仅在当前会话期间有效，关闭浏览器会被清除

  2. 存放数据大小

    • Cookie 不能超过 4K
    • localStorage 一般为 5MB 或者更大
    • sessionStorage 一般为 5MB 或者更大

  3. 与服务器端通信

    • Cookie 每次请求都会携带在 http 头中，如果使用 Cookie 保存过多数据会带来性能问题。在所有同源窗口中都是共享的
    • localStorage 仅在客户端中保存，不参与服务器的通信。在所有同源窗口中都是共享的
    • sessionStorage 不在不同的浏览器窗口共享，即使在同一个页面。仅在客户端中保存，不参与服务器的通信。

  4. 易用性

    • Cookie 需要程序员自己封装，原生的 Cookie 接口不友好
    • localStorage 原生接口可以接受，亦可再次封装来对 Object 和 Array 有更好的支持
    • sessionStorage 原生接口可以接受，亦可再次封装来对 Object 和 Array 有更好的支持

  5. 数据传递不同

    • Cookie 数据始终在同源的 http 请求中携带（即使不需要），即 cookie 在浏览器和服务器间来回传递，cookie 数据还有路径（path）的概念，可以限制 cookie 只属于某个路径下
    • sessionStorage 和 localStorage 不会自动把数据发送给服务器，仅在本地保存。
