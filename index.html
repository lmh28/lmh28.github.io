react组件生命周期

前言：react生命周期构子函数分为三个阶段:挂载阶段、更新阶段、销毁阶段

一、挂载阶段的生命周期函数

  • constructor(props)

    应当做些初始化的动作，如：初始化state，将事件处理函数绑定到类实例上

  • static getDerivedStateFromProps(props,state)

    在组件实例化后,和接受新的props后被调用。
    他必须返回一个对象来更新状态,或者返回null表示新的props不需要任何state的更新。
    如果是由于父组件的props更改，所带来的重新渲染，也会触发此方法

  • render()

    render()方法是必需的。当他被调用时，他将计算this.props和this.state，
    并返回以下一种类型： 

    1. React元素。通过jsx创建，既可以是dom元素，也可以是用户自定义的组件。 
2. 字符串或数字。他们将会以文本节点形式渲染到dom中。 
3. null，什么也不渲染 
4. 布尔值。也是什么都不渲染。
  • componentDidMount()

    组件被装配后立即调用。初始化使得DOM节点应该进行到这里。
    通常在这里进行ajax请求
    如果要初始化第三方的dom库，也在这里进行初始化。
    只有到这里才能获取到真实的dom.

挂载阶段又分为三个过程：
1.将要挂载：componentWillMount
2.执行render方法渲染页面，render方法在挂载阶段会执行
3.挂载完成：componentDidMount（只执行一次）

二、更新阶段的生命周期函数

  • static getDerivedStateFromProps(props, state)
  • shouldComponentUpdate(nextProps, nextState)

    当组件接收新的属性之后(componentWillReceiveProps)
    判断组件是否需要更新：shouldComponentUpdate(默认返回的是true)
    返回true时，表示组件需要更新，会执行：componentWillUpdate将要更新,渲染(render)
    返回false表示后面内容不再执行，不渲染(render)

  • render()
  • getSnapshotBeforeUpdate(prevProps, prevState)

    必须和componentDidUpdate一起使用
    必须返回一个值
    不能和旧版本的钩子函数一起使用
    目的是为了返回数据更新之前的状态
    在react render()后的输出被渲染到DOM之前被调用。它使您的组件能够在它们被潜在更改之前捕获当前值（如滚动位置）。这个生命周期返回的任何值都将作为参数传递给componentDidUpdate（）。

  • componentDidUpdate(prevProps, prevState, snapshot)

    当组件更新时，将此作为一个机会来操作DOM。
    只要您将当前的props与以前的props进行比较
    （例如，如果props没有改变，则可能不需要网络请求），这也是做网络请求的好地方。

在更新阶段我们只要修改了属性(props)和states数据都会重新渲染页面，重新渲染页面就是重新执行render这个函数
注意：如果在render中调用了设置变量或者设置数据的方法，如果在render调用了this.setState方法就会出现死循环这种情况

三、卸载阶段的生命周期函数

  • componentWillUnmount()

    组件被卸载并销毁之前立即被调用。
    在此方法中执行任何必要的清理，
    例如使定时器无效，
    取消网络请求或清理在componentDidMount中创建的任何监听。

js中常见的设计模式

1.单例模式

保证一个类只有一个实例，并提供一个访问它的全局访问点，每一个实例都不相等，都是单一

2.观察者模式(发布-订阅模式)

其定义对象间是一种:一对多的依赖关系，当一个对象的状态发生改变是，所有依赖于它的对象都将得到通知

3.适配器模式

将一个类（对象）的接口（方法或者属性）转化成另外一个接口以满足用户需求，使类（对象）之间接口的不兼容问题通过适配器得以兼容

数据适配

4.组合模式

组合模式又称-整体模式,将对象组合成树形结构以表示’部分整体’的层次结构

移动端开发需要注意的事项

什么是 viewpoint？
viewpoint:视口，默认宽度是 1024，initial-scale 设置比例

1.利用 meta 标签对 viewport 进行控制

该 meta 标签的作用是让当前 viewport 的宽度等于设备的宽度，同时不允许用户手动缩放。也许允不允许用户缩放不同的网站有不同的要求，但让 viewport 的宽度等于设备的宽度，这个应该是大家都想要的效果，如果你不这样的设定的话，那就会使用那个比屏幕宽的默认 viewport，也就是说会出现横向滚动条。

2.用户缩放出现 1px 边框问题

有的手机分辨率比较高，是 2 倍或 3 倍屏，手机上的浏览器就会把 css 中的 1px 放大 2 到 3 倍。可以利用 scroll 缩放的原理将边框重置。

出现 1px 还有用户缩放页面也会引起 css 中 px 的变化。例如：当用户把页面放大或缩小一倍，css 中 1px 所代表的的物理像素也会增加或缩小一倍

3.css 样式统一

我们需要重置页面样式，因为在不同的手机浏览器上，默认的样式是不统一的

4.300 毫秒点击延迟问题

在移动端开发中，某些机型上使用 click 事件会延迟 300ms 才执行，这样影响了用户体验

5.上下拉动滚动条是出现卡顿、慢

6.旋转屏幕时，字体大小调整的问题

常见的定位方式

position:static/relative/absolute/fixed

1.relative:相对定位

相对定位的偏移参考元素是元素本身，不会使定位元素脱离文档流。元素的初始位置占据的空间会被保留。（相对定位常常作为 absolute 的父级定位）。根据 top left bottom right 修改位置

2.absolute:绝对定位

绝对定位元素 以父级元素中最近的定位元素为参考坐标，如果绝对定位元素的父级元素中没有采用定位，那么绝对定位元素的参考对象是根节点(html)，元素会脱离文档流，不占据空间。根据 top left bottom right 修改位置

3.fixed:固定定位

固定定位位移的参考坐标是可视窗口，使用固定定位会使元素脱离文档流。根据 top left bottom right 修改位置

4.static:默认定位

默认值，元素框正常生成的，top left bottom right 这几个偏移属性不会影响其静态定位的正常显示
